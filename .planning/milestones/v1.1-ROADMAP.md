# Milestone v1.1: Microservice Platform

**Status:** ✅ SHIPPED 2026-05-06
**Phases:** 5-6
**Total Plans:** 7

## Overview

Extend the v1.0 CLI into an automated microservice platform that discovers Tier 1 tournament matches, downloads `.dem.gz` replay files, parses them into structured game events, and stores everything in a queryable PostgreSQL database.

## Milestone Summary

**Key Decisions:**
- Single `go.mod` monorepo — services compile alongside v1.0 CLI
- NATS JetStream for service decoupling with WorkQueue retention
- `pgxpool.Pool` (not `pgx.Conn`) with MaxConns=20
- Streaming everywhere — no `io.ReadAll` on demos (50-200MB)
- `INSERT ... ON CONFLICT DO NOTHING` for idempotent writes
- `defer msg.Ack()` mandatory in every NATS handler
- `defer p.Close()` mandatory on every demoinfocs parser
- Python cloudscraper subprocess for Cloudflare challenge bypass
- Firefox uTLS fingerprint + HTTP/1.1 for CDN access
- 7z/unar for RAR extraction before parsing
- MinIO object deletion after successful parse

**Issues Resolved:**
- HTTP/2 vs HTTP/1.1 protocol mismatch on R2 CDN
- Cloudflare Turnstile challenge on demo download page (solved via cloudscraper)
- FK constraint violations (MatchStart not firing for CS2 demos)
- R2 pre-signed URL expiry (fixed via immediate processing)
- Docker network isolation (resolved with native downloader)
- HLTV rate limiting (2s delay between match page fetches)

**Technical Debt Incurred:**
- Downloader runs natively, not in Docker (Docker Desktop VM IP blocked by R2 CDN)
- Parser MapName is empty — demoinfocs-golang v5 doesn't expose it
- Poller fetches all match pages sequentially — could be parallelized

**Verification:**
