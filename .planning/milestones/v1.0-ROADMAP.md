# Milestone v1.0: HLTV CLI MVP

**Status:** ✅ SHIPPED 2026-05-03
**Phases:** 1-4
**Total Plans:** 9

## Overview

Initial MVP of HLTV CLI — an extensible Go command-line tool for retrieving Counter-Strike data from HLTV and emitting JSON for scripts, automations, and downstream tools. Focused on three read-only workflows: listing Tier 1 events, listing completed match results, and returning the demo download link by HLTV match ID.

## Milestone Summary

**Key Decisions:**
- Build in Go with Cobra command framework and goquery HTML parsing.
- JSON-only stdout with `{data, meta}` envelope.
- Structured error JSON on stderr with snake_case codes.
- Provider middleware pattern between commands and infrastructure.
- Functional options pattern for injectable constructors.
- `roundTripFunc` fake transports for sandbox-safe HTTP tests.
- Live HLTV selectors: `[data-demo-link]` (primary), `[data-manuel-download]` (fallback).
- `DisableFlagParsing: true` for `dem demo` (no flags, help via `dem help demo`).

**Issues Resolved:**
- Phase 2 plan 02-02 absorbed into Phase 3 plan 03-01.
- CSS selector staleness resolved during Phase 4 planning with live HLTV HTML.

**Technical Debt Incurred:**
- Pre-existing Phase 3 provider tests need fixture updates (fail on modified events.html/results.html).
- `httptest.NewServer` blocked by sandbox — all tests use `roundTripFunc`.
- Phase 2 UAT incomplete (1/9 passed, 8 pending).
