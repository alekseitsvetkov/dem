# Requirements: HLTV CLI — v1.1 Microservice Platform

**Defined:** 2026-05-03
**Shipped:** 2026-05-06
**Status:** ✅ All requirements validated

## Infrastructure

- [x] **INFR-01**: Developer can start all infrastructure with a single `docker-compose up` command.
- [x] **INFR-02**: Go monorepo uses a single root `go.mod`.
- [x] **INFR-03**: Database schema managed via migrations, 8 tables created.
- [x] **INFR-04**: Shared packages (pkg/natsutil, pkg/minio, pkg/postgres) with functional options.
- [x] **INFR-05**: NATS JetStream streams created programmatically at service startup.
- [x] **INFR-06**: Postgres connection pooling uses pgxpool with MaxConns.

## Tournament Polling

- [x] **POLL-01**: Poller checks HLTV for Tier 1 tournaments on configurable schedule, reuses v1.0 parsers.
- [x] **POLL-02**: Poller discovers matches with available demos and publishes download jobs to NATS.
- [x] **POLL-03**: Poller deduplicates matches via processed_matches table.

## Demo Download

- [x] **DWLD-01**: Downloader consumes jobs from NATS with JetStream pull consumer and explicit Ack.
- [x] **DWLD-02**: Downloader streams .dem.gz from HLTV CDN to MinIO without local disk writes.
- [x] **DWLD-03**: On success, publishes parse job to NATS; on failure, NakWithDelay.

## Demo Parsing

- [x] **PARS-01**: Parser consumes jobs from NATS with JetStream pull consumer.
- [x] **PARS-02**: Parser streams .dem.gz from MinIO into demoinfocs-golang.
- [x] **PARS-03**: Parser registers 12 event handlers for all required game events.
- [x] **PARS-04**: Parser batch-inserts game events with idempotent ON CONFLICT (event_id) DO NOTHING.
- [x] **PARS-05**: Parser calls defer p.Close() on every demoinfocs instance.

## Cross-Cutting

- [x] **CROS-01**: Every service uses structured logging via log/slog with correlation fields.
- [x] **CROS-02**: All external dependencies injected via functional options.
- [x] **CROS-03**: Existing v1.0 code is not modified — imported as library only.
