<threat_model>
## Security Threat Model

**Scope:** EventsProvider layer, events CLI command, error mapping to JSON envelopes.

| Threat | STRIDE | Severity | Disposition | Mitigation |
|--------|--------|----------|-------------|------------|
| Validation executed after network access (D-09 violation) | Tampering | High | Mitigate | Validate `--tier` non-empty and `--limit >= 0` at the top of RunE, before any provider/network call |
| ProviderError or ParseError leaks internal state in Details() | Information Disclosure | Medium | Mitigate | Type switch extracts `.Code` and `.Details()` only; no stack traces, no raw response bodies |
| Tier filter is case-sensitive (user passes "1" but HLTV has "1") | Spoofing | Low | Mitigate | `filterByTier` uses `strings.EqualFold` for case-insensitive comparison |
| Fake RoundTripper in tests accidentally uses real transport | Spoofing | Medium | Mitigate | Tests construct `hltv.Client` with `WithHTTPClient(&http.Client{Transport: transport})` — no default transport |
| --limit negative value causes slice panic | Denial of Service | Medium | Mitigate | Validate `limit >= 0` before provider call; provider checks `limit > 0` before slicing |
</threat_model>

       ```go
       import (
           "bytes"
           "context"

           "github.com/alekseitsvetkov/dem/internal/domain"

       // EventsProvider is the interface for fetching and filtering events.
       type EventsProvider interface {
           GetEvents(ctx context.Context, tier string, limit int) ([]domain.Event, error)
       }

       type eventsProvider struct {
           client *hltv.Client
           urls   hltv.URLs
       }

       type EventsProviderOption func(*eventsProvider)

       func NewEventsProvider(opts ...EventsProviderOption) EventsProvider {
           p := &eventsProvider{
               client: hltv.NewClient(),
               urls:   hltv.NewURLs(""),
           }
           for _, opt := range opts {
               opt(p)
           }
           return p
       }

       func WithEventsClient(c *hltv.Client) EventsProviderOption {
           return func(p *eventsProvider) { p.client = c }
       }

       func WithEventsURLs(u hltv.URLs) EventsProviderOption {
           return func(p *eventsProvider) { p.urls = u }
       }

       func (p *eventsProvider) GetEvents(ctx context.Context, tier string, limit int) ([]domain.Event, error) {
           body, err := p.client.Fetch(ctx, p.urls.EventsURL())
           if err != nil {
               return nil, err // Pass through ProviderError without remapping (D-03, D-08)
           }

           events, err := parser.ParseEvents(bytes.NewReader(body), p.urls.EventsURL())
           if err != nil {
               return nil, err // Pass through ParseError without remapping
           }

           if tier != "" {
               events = filterByTier(events, tier)
           }
           if limit > 0 && limit < len(events) {
               events = events[:limit]
           }

           return events, nil
       }
       ```

       Add a package-level helper `filterByTier` (not exported — implementation detail):

       ```go
       func filterByTier(events []domain.Event, tier string) []domain.Event {
           tier = strings.TrimSpace(tier)
           if tier == "" {
               return events
           }
           var filtered []domain.Event
           for _, e := range events {
               if strings.EqualFold(strings.TrimSpace(e.Tier), tier) {
                   filtered = append(filtered, e)
               }
           }
           return filtered
       }
       ```

       Key decisions:
       - `NewEventsProvider` returns the `EventsProvider` interface (not a concrete pointer). Callers depend on the interface only.
       - Option constructors `WithEventsClient` and `WithEventsURLs` follow the Phase 2 `hltv.WithHTTPClient` naming pattern.
       - Provider errors pass through unchanged per D-03/D-08. The command handler type-asserts them for error mapping.
       - `filterByTier` is case-insensitive (EqualFold), trimmed on both sides.
       - A limit of 0 or negative means no truncation. Only positive limits with fewer items than the slice trigger truncation.

       ```go
       package cli

       import (
           "context"
           "io"
           "github.com/alekseitsvetkov/dem/internal/output"
           "github.com/alekseitsvetkov/dem/internal/provider"
           "github.com/spf13/cobra"
       )

       func newEventsCommand(out io.Writer, errOut io.Writer, p provider.EventsProvider) *cobra.Command {
           cmd := &cobra.Command{
               Use:   "events",
               Short: "List HLTV events as JSON",
               Long: "Fetch events from HLTV, optionally filter by tier, and return them as a JSON array. " +
                   "When --tier is omitted, all events are returned unfiltered. " +
                   "When --limit is 0 (default), all events from the page are returned.",
               RunE: func(cmd *cobra.Command, args []string) error {
                   tier, _ := cmd.Flags().GetString("tier")
                   limit, _ := cmd.Flags().GetInt("limit")

                   // D-09: Validation before any network access
                   if limit < 0 {
                       return output.WriteErrorJSON(errOut, "validation_error",
                           "--limit must be greater than or equal to 0",
                           map[string]any{"flag": "limit", "value": limit})
                   }

                   events, err := p.GetEvents(cmd.Context(), tier, limit)
                   if err != nil {
                       return mapEventsError(errOut, err)
                   }

                   if events == nil {
                       events = []domain.Event{} // ensure JSON array, not null
                   }
                   return output.WriteJSON(out, events, nil)
               },
           }

           cmd.Flags().String("tier", "", "Filter events by tier (e.g., \"1\", \"S-Tier\")")
           cmd.Flags().Int("limit", 0, "Maximum number of events to return (0 = no limit)")
           return cmd
       }
       ```

       Add a helper function `mapEventsError` in the same file:

       ```go
       func mapEventsError(w io.Writer, err error) error {
           switch e := err.(type) {
           case *hltv.ProviderError:
               return output.WriteErrorJSON(w, e.Code, e.Message, e.Details())
           case *parser.ParseError:
               return output.WriteErrorJSON(w, e.Code, e.Message, e.Details())
           default:
               return output.WriteErrorJSON(w, "internal_error", err.Error(), nil)
           }
       }
       ```

       Import paths needed: `"github.com/alekseitsvetkov/dem/internal/domain"`, `"github.com/alekseitsvetkov/dem/internal/hltv"`, `"github.com/alekseitsvetkov/dem/internal/hltv/parser"`.

       Key decisions:
       - `--tier` is optional (default ""). When empty, all events are returned unfiltered. The provider-level `filterByTier` function only filters when tier is non-empty.
       - `--limit` defaults to 0 (no limit). Validation checks `limit >= 0`. The provider truncates only when `limit > 0 && limit < len(events)`.
       - `mapEventsError` is file-local (unexported) and handles both error types with a type switch. This avoids needing a common error interface. All three cases call `output.WriteErrorJSON` with the code, message, and structured details.
       - When events is nil, coerce to empty slice so JSON is `[]` not `null`.
