`matches` table: match_id TEXT PK, team1/team2 TEXT NOT NULL, map_name TEXT, event_name TEXT, match_date DATE, tick_rate DOUBLE PRECISION, duration_secs DOUBLE PRECISION, demo_url TEXT, minio_key TEXT, parsed_at TIMESTAMPTZ DEFAULT NOW(), created_at TIMESTAMPTZ DEFAULT NOW()

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Docker host network -> Postgres port 5432 | Exposed database port on localhost |
| Docker host network -> NATS port 4222 | Exposed message broker on localhost |
| Docker host network -> MinIO ports 9000/9001 | Exposed object storage and console (credentials: minioadmin/minioadmin) |
| Docker host network -> Redis port 6379 | Exposed cache (no auth by default) |
| Container -> Container (internal docker network) | Inter-service communication within docker network |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-01 | S | Postgres (docker-compose.yml) | accept | Development-only credentials (dem/dem). Production would require vault-managed secrets. No PII stored. |
| T-05-02 | S | MinIO (docker-compose.yml) | accept | Development-only credentials (minioadmin/minioadmin). S3 access in dev network only. |
| T-05-03 | T | Redis (docker-compose.yml) | accept | No authentication configured — acceptable for local dev only. Port only on localhost. |
| T-05-04 | I | sql/migrations DDL | mitigate | Migration UP files represent authoritative schema. DOWN files are destructive (CASCADE) — applied in controlled dev only, never in production without review. |
| T-05-05 | E | Docker volumes (pgdata, miniodata, redisdata) | accept | Named Docker volumes persist data across container restarts. No sensitive data at this stage. |
</threat_model>
