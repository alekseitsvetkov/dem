**Run method implementation:**

1. **Cron setup (D-01):** Use `robfig/cron/v3` (`cron.New(cron.WithSeconds())`). Add a single job with `p.cfg.CronExpression`. The job calls `p.poll(ctx)`.

2. **Minimum interval guard (D-01):** Before each poll run, check if the last successful run was less than `p.cfg.MinInterval` ago (store `lastRun time.Time` in PollerService, protected by `sync.Mutex`). If too soon, log a warning and skip.

3. **poll(ctx) method — the core discovery logic:**
   a. Fetch events page: `body, err := p.client.Fetch(ctx, p.urls.EventsURL())`
   b. Parse events: `events, err := parser.ParseEvents(bytes.NewReader(body), p.urls.EventsURL())`
   c. Filter Tier 1: replicate v1.0 filterTier1 logic — keep events where `PrizePool > 250_000` OR name case-insensitive contains any of: `"IEM", "PGL", "Blast", "StarLadder", "FISSURE", "Esports World Cup", "Major", "BetBoom"`
   d. For each Tier 1 event:
      - Extract numeric event ID from `event.ID`
      - Fetch results: `body, _ := p.client.Fetch(ctx, p.urls.ResultsURLForEvent(eventID))`
      - Parse results: `results, _ := parser.ParseResults(bytes.NewReader(body), resultsURL)`
      - For each result:
        - Convert `result.MatchID` to int64
        - Fetch match page: `body, _ := p.client.Fetch(ctx, p.urls.MatchURL(matchIDInt))`
        - Parse demo link: `demoLink, err := parser.ParseDemoLink(bytes.NewReader(body), matchURL)`
        - If `err != nil` and error code is `ErrorCodeUnavailableData` — skip (no demo available)
        - If `demoLink.DemoURL != ""`:
          - Dedup: `INSERT INTO processed_matches (match_id) VALUES ($1) ON CONFLICT DO NOTHING`, use `pool.Exec(ctx, query, matchIDInt)`; check `cmd.RowsAffected()`. If 0, skip (already processed per D-02).
          - Publish NATS message to `SubjectDownload` (`"dem.download.jobs"`):
            JSON payload: `{"match_id": "<matchID>", "demo_url": "<demoLink.DemoURL>", "match_url": "<demoLink.MatchURL>", "event_name": "<event.Name>", "team1": "<result.Team1>", "team2": "<result.Team2>", "match_date": "<result.Date>", "discovered_at": "<time.Now().UTC().Format(time.RFC3339)>"}`
            Use `js.PublishMsg()` with `nats.Msg{Subject: natsutil.SubjectDownload, Data: jsonBytes}`
          - Log: `p.logger.Info("published download job", slog.String("match_id", matchID), slog.String("event", event.Name))`

4. **Error handling:** Individual fetch/parse failures should be logged and continue to the next event/result. The cron job should not abort on a single failing match. Use `p.logger.Warn(...)` for transient errors. Use `p.logger.Error(...)` for persistent failures.

5. **Log format (CROS-01):** Every log line uses `slog.String("match_id", ...)` for match-level context. Top-level logs use `slog.String("event_id", ...)`. No `{data, meta}` JSON envelopes.

6. **CROS-03 compliance:** Only import from `internal/hltv` (Client, URLs, parser/*) and `internal/domain` (types). Do NOT import from `internal/provider/`, `internal/cli/`, `internal/output/`, or `cmd/dem/`.

If `robfig/cron/v3` is not in go.mod, add `github.com/robfig/cron/v3 v3.0.1` to the require block and run `go mod tidy`.

**Concrete implementation:**

```go
package main

import (
    "context"
    "log/slog"
    "os"

    "github.com/alekseitsvetkov/dem/internal/hltv"
    "github.com/alekseitsvetkov/dem/internal/poller"
    "github.com/alekseitsvetkov/dem/internal/service"
    "github.com/alekseitsvetkov/dem/pkg/natsutil"
    "github.com/alekseitsvetkov/dem/pkg/postgres"
)

func main() {
    cfg := poller.LoadConfig()

    logger := slog.New(slog.NewJSONHandler(os.Stdout, &slog.HandlerOptions{
        Level: parseLogLevel(cfg.LogLevel),
    }))

    // Wire NATS
    nc, js, err := natsutil.NewNATSConn(
        natsutil.WithURL(cfg.NATSURL),
        natsutil.WithName("poller"),
    )
    if err != nil {
        logger.Error("nats connect failed", slog.String("error", err.Error()))
        os.Exit(1)
    }
    defer nc.Close()

    // Wire Postgres
    pool, err := postgres.NewPool(context.Background(), cfg.DatabaseURL)
    if err != nil {
        logger.Error("postgres connect failed", slog.String("error", err.Error()))
        os.Exit(1)
    }
    defer pool.Close()

    // Wire HLTV Client (reuse v1.0 client — per D-01)
    hltvClient := hltv.NewClient()

    // Create PollerService with functional options (per CROS-02)
    svc := poller.NewPollerService(cfg,
        poller.WithNATS(js),
        poller.WithPostgres(pool),
        poller.WithHLTVClient(hltvClient),
        poller.WithLogger(logger),
    )

    // Create Runner and add service
    runner := service.NewRunner(
        service.WithLogger(logger),
    )
    runner.AddService(svc)

    if err := runner.Run(context.Background()); err != nil {
        logger.Error("poller service exited with error", slog.String("error", err.Error()))
        os.Exit(1)
    }
}

func parseLogLevel(level string) slog.Level {
    switch level {
    case "debug":
        return slog.LevelDebug
    case "warn":
        return slog.LevelWarn
    case "error":
        return slog.LevelError
    default:
        return slog.LevelInfo
    }
}
```

- `parseLogLevel` helper for configurable log level
- v1.0 code not imported beyond `internal/hltv` (CROS-03)

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| HLTV.org -> Poller | Untrusted HTML from public website crosses into the process via `Client.Fetch` |
| Poller -> NATS | Download job payloads cross into the messaging system |
| Poller -> Postgres | Deduplication queries cross into the database |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-04 | Spoofing | HLTV HTML content | mitigate | The v1.0 goquery parser extracts structured fields from known CSS selectors. Crafted HTML that matches selectors but contains garbage data produces garbage jobs — but the downloader URL validation and parser idempotency prevent downstream corruption. |
| T-06-05 | Denial of Service | Poller cron loop | mitigate | Minimum interval guard (default 6h, D-01) prevents runaway polling. `robfig/cron` v3 runs one job at a time (`cron.WithSeconds()` with single-function entry prevents concurrent poll runs). |
| T-06-06 | Information Disclosure | NATS message payload | accept | Download job payloads contain only public HLTV data (match_id, team names, event name, demo URL). No secrets, no PII. Transmitted over localhost NATS by default. |
| T-06-07 | Elevation of Privilege | Postgres SQL injection | mitigate | All Postgres queries use parameterized `$1` placeholders with `pgxpool.Pool.Exec`. Match IDs are parsed as int64 before use. No string concatenation in SQL. |
</threat_model>
