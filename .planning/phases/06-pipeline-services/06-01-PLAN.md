  <action>
Create `internal/service/runner.go` with the following concrete API surface:

**Package:** `service` (package declaration is `package service`)

**Exported types and functions:**

1. `type Service interface { Run(ctx context.Context) error }` — the contract every pipeline service implements. The `ctx` is cancelled when the Runner receives SIGTERM/SIGINT, so services can use it for graceful shutdown of their own loops.

2. `type Runner struct` — unexported fields: `services []Service`, `logger *slog.Logger`, `shutdownTimeout time.Duration`

3. `type RunnerOption func(*Runner)` — functional option type

4. `func NewRunner(opts ...RunnerOption) *Runner` — constructor with defaults: `logger: slog.Default()`, `shutdownTimeout: 30 * time.Second`. Applies opts.

5. `func WithLogger(logger *slog.Logger) RunnerOption` — sets the structured logger. Default is `slog.Default()`.

6. `func WithShutdownTimeout(d time.Duration) RunnerOption` — sets the max time to wait for services to stop after signal. Default: `30 * time.Second`.

7. `func (r *Runner) AddService(svc Service)` — appends a service to the runner. Services are started in the order added and stopped in reverse order.

8. `func (r *Runner) Run(ctx context.Context) error` — the main entry point:
   - Creates a signal-aware context via `signal.NotifyContext(context.Background(), syscall.SIGTERM, syscall.SIGINT)`
   - Logs "starting services" with count via `r.logger.Info("starting services", slog.Int("count", len(r.services)))`
   - Starts each service in a separate goroutine, capturing errors via an error channel (`make(chan error, len(r.services))`)
   - Waits for either: a service error, or the signal context being done
   - On shutdown (signal received or service error): logs "shutting down", creates a context with `r.shutdownTimeout`, calls `svc.Run(shutdownCtx)` for each service in **reverse** order, waiting for each to complete
   - Returns the first error encountered, or nil if all services started and stopped cleanly

Imports needed: `context`, `log/slog`, `os/signal`, `syscall`, `sync`, `time`.

Do NOT import any NATS, MinIO, or Postgres packages — this package is pure stdlib + slog.
Do NOT modify any existing files outside this package.
