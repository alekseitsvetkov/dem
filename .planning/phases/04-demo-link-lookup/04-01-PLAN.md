    **Function `extractMatchID(matchURL string) string`:**

    Parses an HLTV match URL to extract the numeric match ID.
    - Example: `https://www.hltv.org/matches/107224/esl-pro-league-season-21` returns `"107224"`
    - Parse URL with `url.Parse(matchURL)`, split path segments by `/`
    - If path has at least 2 segments and the first segment (after trimming `/`) is `"matches"`, return the second segment
    - Fallback: return `path.Base(matchURL)` (mirrors `extractEventID` pattern)
    - On URL parse error, return `""`

    **Function `ParseDemoLink(r io.Reader, matchURL string) (domain.DemoLink, error)`:**

    Follows the established parser pattern from `ParseEvents`/`ParseResults` (accept `io.Reader`, returns typed domain model + error).

    1. **Parse HTML:** Call `goquery.NewDocumentFromReader(r)`
       - On error: return `(domain.DemoLink{}, &ParseError{Code: ErrorCodeParse, Area: "demo", Err: err})`
    2. **Extract matchID:** Call `extractMatchID(matchURL)` — always succeeds even if empty
    3. **Primary selector:** `doc.Find("[data-demo-link]")`
       - If `sel.Length() > 0`: extract `data-demo-link` attribute via `sel.First().Attr("data-demo-link")`
       - If attribute exists and is non-empty: resolve via `resolveURL(href)`, return `DemoLink{MatchID, MatchURL, DemoURL}` with nil error
       - If attribute empty or missing: fall through to fallback selector
    4. **Fallback selector:** `doc.Find("[data-manuel-download]")`
       - If `sel.Length() > 0`: extract `href` attribute via `sel.First().Attr("href")`
       - If attribute exists and is non-empty: resolve via `resolveURL(href)`, return `DemoLink{MatchID, MatchURL, DemoURL}` with nil error
       - If attribute empty or missing: fall through to unavailable_data
    5. **Unavailable data:** Return partial `DemoLink{MatchID: matchID, MatchURL: matchURL}` with:
       ```go
       &ParseError{
           Code:    ErrorCodeUnavailableData,
           Area:    "demo",
           Message: "no demo available for this match",
       }
       ```
    6. **Godoc comment** (per Pitfall 3 from research): Document that ParseDemoLink returns BOTH a partial DemoLink AND a non-nil error when no demo is available. The caller must check the error AND can use the partial DemoLink for context.

    **Key decisions per user decisions:**
    - Per D-03 and the live HTML structure: use `[data-demo-link]` as primary selector (NOT `a.demo-link` — that class is stale)
    - `resolveURL` is already defined in `events.go` in the same package — callable directly without import
    - MatchID type: `string` (matches `domain.DemoLink.MatchID` which is `string`; the int conversion happens in the provider layer)

| T-04-03 | Information Disclosure | Match URL extraction | accept | Match URL is constructed from a user-provided match ID, not parsed from external HTML; no PII involved |
</threat_model>
