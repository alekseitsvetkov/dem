    **GetDemo method (per D-03, D-04, D-05/D-06):**

    ```go
    func (p *demoProvider) GetDemo(ctx context.Context, matchID int) (domain.DemoLink, error) {
        matchURL := p.urls.MatchURL(matchID)

        body, err := p.client.Fetch(ctx, matchURL)
        if err != nil {
            return domain.DemoLink{}, err // Pass through ProviderError without remapping (D-08)
        }

        link, err := parser.ParseDemoLink(bytes.NewReader(body), matchURL)
        if err != nil {
            var parseErr *parser.ParseError
            if errors.As(err, &parseErr) && parseErr.Code == parser.ErrorCodeUnavailableData {
                // D-03: Unavailable data → return success with partial DemoLink
                return link, nil
            }
            return domain.DemoLink{}, err // Pass through other ParseErrors
        }

        return link, nil
    }
    ```

    Key design decisions:
    - Per D-03: `unavailable_data` from parser is converted to success (nil error) with partial DemoLink
    - Per D-08: Use `errors.As` with `*parser.ParseError` for type-safe error check (not type switch — we specifically want to handle `unavailable_data` differently from other parse errors)
    - Per Phase 3 D-08/Phase 1 D-07: Other ProviderError/ParseError values pass through unmapped; the CLI command handler maps them to JSON error envelopes
    - MatchURL is built from matchID using existing URLs.MatchURL

    **Godoc on GetDemo:** Document that when no demo is available, the method returns a DemoLink with DemoURL empty (zero value) and nil error. The caller can check `link.DemoURL == ""` to detect unavailable demos.

    **Function `newDemoCommand(out io.Writer, errOut io.Writer, p provider.DemoProvider) *cobra.Command`**

    ```go
    func newDemoCommand(out io.Writer, errOut io.Writer, p provider.DemoProvider) *cobra.Command {
        cmd := &cobra.Command{
            Use:                "demo <match-id>",
            Short:              "Get demo download link for an HLTV match",
            Long: "Fetch an HLTV match page and return the demo download link as JSON. " +
                "When a demo is available, the response includes demo_url. " +
                "When no demo is available, demo_url is omitted (exit code 0). " +
                "Use 'dem help demo' for help (--help does not work on this command).",
            Args:               cobra.ExactArgs(1),
            DisableFlagParsing: true, // D-07: zero-flag command; prevents cobra from parsing args as flags
            RunE: func(cmd *cobra.Command, args []string) error {
                // D-05, D-06: Validation before any network access
                matchID, err := strconv.Atoi(args[0])
                if err != nil || matchID <= 0 {
                    _ = output.WriteErrorJSON(errOut, "validation_error",
                        "match-id must be a positive integer",
                        map[string]any{"arg": args[0]})
                    return fmt.Errorf("invalid match-id: %q", args[0])
                }

                link, err := p.GetDemo(cmd.Context(), matchID)
                if err != nil {
                    _ = mapDemoError(errOut, err)
                    return err
                }

                return output.WriteJSON(out, link, nil)
            },
        }
        return cmd
    }
    ```

    Key design decisions:
    - `DisableFlagParsing: true` per D-07 (zero flags). `--help` is NOT available on this command; users use `dem help demo`.
    - `cobra.ExactArgs(1)` ensures exactly one positional argument
    - Validation with `strconv.Atoi(...) && matchID <= 0` per D-06 — catches both non-numeric input AND zero/negative
    - `output.WriteJSON(out, link, nil)` — the DemoLink struct's `omitempty` on DemoURL means the field is absent from JSON when empty (per D-04)
    - Long description documents the help behavior tradeoff per Pitfall 1 from research

    **Function `mapDemoError(w io.Writer, err error) error`**
    Follows the exact pattern from `mapEventsError` and `mapResultsError`:

    ```go
    func mapDemoError(w io.Writer, err error) error {
        switch e := err.(type) {
        case *hltv.ProviderError:
            return output.WriteErrorJSON(w, e.Code, e.Message, e.Details())
        case *parser.ParseError:
            return output.WriteErrorJSON(w, e.Code, e.Message, e.Details())
        default:
            return output.WriteErrorJSON(w, "internal_error", err.Error(), nil)
        }
    }
    ```
