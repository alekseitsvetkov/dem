<threat_model>
## Security Threat Model

**Scope:** Error handling, process exit behavior, and CLI contract tests.

| Threat | Severity | Mitigation |
|--------|----------|------------|
| Errors print plain text or usage to stderr, breaking JSON-only automation assumptions. | high | Configure Cobra to silence default usage/errors and write centralized JSON errors through `output.WriteErrorJSON`. |
| Validation failures accidentally make network calls in later commands. | medium | Establish tests and handler pattern where validation happens before dependencies are called. |
| Error details leak sensitive values in future commands. | medium | Use explicit `details` maps; do not automatically dump raw errors or environment data. |
</threat_model>
