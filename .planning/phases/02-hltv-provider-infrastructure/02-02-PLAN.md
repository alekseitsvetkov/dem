<threat_model>
## Security Threat Model

**Scope:** HTML parser package, checked-in fixtures, and domain models.

| Threat | Severity | Mitigation |
|--------|----------|------------|
| Parser tests accidentally fetch live HLTV pages. | high | Use only files under `internal/hltv/parser/testdata`; tests must not import `net/http`. |
| Raw HLTV pages include bulky scripts, trackers, or unrelated content in the repository. | medium | Store trimmed sanitized fixtures containing only relevant page regions and class/attribute structure. |
| Parser errors leak raw HTML into structured details. | medium | Parser errors expose code, area, field, and message only; never include full fixture/page content in error details. |
| Missing demo links are misclassified as parser breakage. | medium | Return `unavailable_data` for valid match pages without demo links. |
| Command handlers later duplicate selectors. | medium | Keep all selectors in `internal/hltv/parser` and add a verification grep against `internal/cli` selector imports. |
</threat_model>
