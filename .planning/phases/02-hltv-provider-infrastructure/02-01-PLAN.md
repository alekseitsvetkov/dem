<threat_model>
## Security Threat Model

**Scope:** Public HLTV HTTP fetching, URL construction, and provider error values.

| Threat | Severity | Mitigation |
|--------|----------|------------|
| Requests hang indefinitely and make CLI automation stall. | medium | Define `DefaultTimeout = 15 * time.Second`, apply it in the default client, and test that the default is configured. |
| The CLI sends anonymous or misleading HTTP requests. | medium | Define `DefaultUserAgent = "dem/dev"` and set it on every request unless explicitly overridden. |
| Provider errors leak raw HTML response bodies into command error details. | medium | Provider errors should carry URL, code, status code, and message fields only; do not store raw response bodies in error details. |
| Tests depend on live HLTV availability. | high | Use `httptest.Server` and fake transports only; no test should call `https://www.hltv.org`. |
| Automatic retries accidentally hammer HLTV when pages fail. | medium | Do not implement retry loops in Phase 2. |
</threat_model>
